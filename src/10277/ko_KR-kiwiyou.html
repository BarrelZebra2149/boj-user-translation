누리온

문제

한국 슈퍼컴퓨터 제5호기 누리온은 564,740 개의 코어를 자랑하며 top500 랭킹에서 46위에 올라 있다. 슈퍼컴퓨터는 매우 많은 전력을 소모하기 때문에, 사용하지 않는 코어는 작동 성능을 낮추어 전력 소모를 줄이려고 한다. 작업을 각 코어에 배정하는 스케줄러는 다음 명령들을 통해 성능을 조정한다.

  • change X S 코어 X의 성능 값을 S만큼 더한다.
  • groupchange A B S 구간 [A, B]에 속하는 모든 코어의 성능 값을 S만큼 더한다.
  • state X 코어 X의 성능을 출력한다.

나중에도 이 프로그램을 사용할 수 있게끔 여러분은 4 587 520 개의 코어를 다룰 수 있어야 한다. 각 코어의 초기 성능은 0이다.

입력

첫째 줄에 코어의 수 C, 각 코어의 최대 성능 N, 명령의 수 O가 주어진다. (1 ≤ C ≤ 4 587 520; 1 ≤ N ≤ 10 000; 1 ≤ O ≤ 50 000) 다음 O 개의 줄에 설명과 같이 명령들이 주어진다. X, A and B는 코어의 번호로, 0부터 시작한다. (0 ≤ A, B, X < C; A ≤ B). S는 성능 변화를 나타내는 정수이다. (−N ≤ S ≤ +N).

코어의 성능은 0을 넘어서 작아지지 않고, N을 넘어서 커지지 않는다. groupchange 명령의 경우 코어의 성능 값을 1씩 더하거나 빼다가, 구간에 속한 어느 하나의 코어의 성능이 더 이상 작아지거나 커질 수 없다면 성능 변화를 멈춘다.

출력

명령마다 한 줄에 결과를 출력한다. change와 groupchange 명령의 경우 실제 성능 값의 변화를 나타내는 정수를 출력한다. state 명령의 경우 코어의 현재 성능을 출력한다.
