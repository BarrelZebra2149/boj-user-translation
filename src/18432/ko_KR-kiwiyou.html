JJOOII 2

문제

비타로는 생일 선물로 길이 N의 문자열 S를 받았다. 문자열은 J, O I 세 가지 문자로 구성된다.

양의 정수 K에 대해서, 문자 J가 K개, 문자 O가 K개, 문자 I가 K개 순서대로 있는 문자열은 K단계 JOI 문자열이라고 한다. 예를 들어 JJOOII는 2단계 JOI 문자열이다.

비타로는 K단계 JOI 문자열을 좋아해서 문자열 S에 아래 연산을 원하는 순서대로 원하는 만큼 수행해 K단계 JOI 문자열로 만들려고 한다.

  • 연산 1 S의 첫 글자를 지운다.
  • 연산 2 S의 마지막 글자를 지운다.
  • 연산 3 S의 처음도 마지막도 아닌 글자를 지운다.

연산 3을 수행하는 데는 시간이 많이 걸리기 때문에 비타로는 연산 3을 최소로 수행해여 K단계 JOI 문자열을 만들려고 한다.

길이 N의 문자열 S와 양의 정수 K가 주어질 때 S를 K단계 JOI 문자열로 만들기 위해 필요한 연산 3의 최소 횟수를 구하자. 만약 연산을 통해 K단계 JOI 문자열을 만들 수 없다면 -1을 대신 출력한다.

입력

다표준 입력에서 아래 데이터를 읽는다. N과 K는 정수이다. S는 문자열이다. 

N K
S

출력

표준 출력에 S를 K단계 JOI 문자열로 만들기 위해 필요한 연산 3의 최소 횟수를 출력한다. 만약 K단계 JOI 문자열을 만들 수 없다면 -1을 대신 출력한다.

제한

  • 3 ≤ N ≤ 200 000.
  • 1 ≤ K ≤ N/3.
  • S는 J, O, I로만 구성된 길이 N의 문자열이다.

추가 제약 조건 없음.

다음과 같이 S를 K단계 JOI 문자열로 만들 수 있다.

  1. 연산 1을 수행하면 S는 JIJOIOIIJ가 된다.
  2. 연산 2를 수행하면 S는 JIJOIOII가 된다.
  3. 연산 3을 수행하여 2번째 글자를 지우면 S는 JJOIOII가 된다.
  4. 연산 3을 수행하여 4번째 글자를 지우면 S는 JJOOII가 된다.

연산 3을 2번보다 적게 수행하여 K단계 JOI 문자열을 만들 수 없으므로 2를 출력한다.

연산을 수행할 필요가 없다.

S를 1단계 JOI 문자열로 만들 수 없다.
